GOAL:
Diagnose a failing test quickly and separate product bug from bad test setup.

CONTEXT:
Describe what changed before the failure appeared.

FAILING TEST:
Paste the test name and the smallest failing snippet.

ERROR OUTPUT:
Paste the exact assertion failure or traceback.

RELATED CODE:
Paste only the production code directly involved.

TASK:
1. Determine whether the test or the implementation is wrong.
2. Explain why.
3. Suggest the minimal fix.
4. Mention any additional test cases needed.

CONSTRAINTS:
- keep output concise
- avoid broad refactors
- preserve intended behavior

OUTPUT:
- diagnosis
- minimal fix
- extra tests
